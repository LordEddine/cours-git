Gestion des conflits

Un conflit survient quand deux branches modifient la même ligne du même fichier.

⚠️ Fichier en conflit

# config.py - Fichier en conflit

<<<<<<< HEAD
DATABASE_URL = "localhost:5432"
=======
DATABASE_URL = "staging.db:5432"
>>>>>>> feature-staging

HEAD = version actuelle | Autre = version à merger

✅ Résolution

  1. 1 Ouvrir le fichier conflictuel
  2. 2 Choisir ou combiner les versions
  3. 3 Supprimer les marqueurs <<<<
  4. 4 Sauvegarder et valider
$ git add config.py
$ git commit -m "fix: résout conflit DB"
