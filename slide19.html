Forks et Contributions Open Source

Un fork est une copie complète d'un dépôt sous votre compte GitHub.

Workflow de contribution

  1. 1 Fork le projet sur GitHub
  2. 2 Clone votre fork en local
  3. 3 Créer une branche de travail
  4. 4 Faire vos modifications + commit
  5. 5 Push vers votre fork
  6. 6 Créer une Pull Request vers l'original

Synchroniser avec l'original

# Ajouter l'original comme "upstream"
$ git remote add upstream [URL_ORIGINAL]

# Récupérer les mises à jour
$ git fetch upstream
$ git merge upstream/main
