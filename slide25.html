Git Rebase

Réorganiser les commits en déplaçant votre branche sur une nouvelle base.

Merge vs Rebase

Merge : Crée un commit de fusion

Historique non-linéaire, mais préserve l'histoire

Rebase : Réécrit l'historique

Historique linéaire, plus propre

$ git checkout feature
$ git rebase main

⚠️ Règles d'or

  • ✅ Rebase vos branches locales
  • ✅ Avant de merger dans main
  • ✗ JAMAIS sur branches partagées
  • ✗ Après un push public
# Rebase interactif
$ git rebase -i HEAD~3
Modifier les 3 derniers commits
