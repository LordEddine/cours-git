Workflow recommandé

Un processus efficace pour travailler avec Git au quotidien.

1. Créer branche
→
2. Développer
→
3. Commits atomiques
→
4. Push + PR
→
5. Review + Merge

✅ À faire

  • ✓ Commits fréquents et atomiques
  • ✓ Messages descriptifs
  • ✓ Pull avant de push
  • ✓ Branches pour chaque feature
  • ✓ Revue de code via PR

✗ À éviter

  • ✗ Commiter tout d'un coup
  • ✗ Messages vagues ("fix", "update")
  • ✗ Force push sur main
  • ✗ Travailler directement sur main
  • ✗ Ignorer les conflits
