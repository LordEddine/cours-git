Pull Requests (PR)

Une PR propose d'intégrer les changements d'une branche dans une autre avec revue de code.

Processus

  1. 1 Push votre branche vers GitHub
  2. 2 Cliquer "New Pull Request"
  3. 3 Décrire les changements
  4. 4 Demander une review
  5. 5 Merger après approbation
$ git push -u origin feature-api

Bonnes pratiques PR

  • ✅ Titre clair et descriptif
  • ✅ Description détaillée des changements
  • ✅ PR de taille raisonnable (≤ 400 lignes)
  • ✅ Tests passés avant review
  • ✅ Répondre aux commentaires

Squash and Merge : Condense tous les commits en un seul
